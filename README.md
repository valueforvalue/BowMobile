# Bow Mobile — Canon Copier Parts Cross-Reference (Android)

**Bow Mobile** is the Android front-end for the Canon Copier Parts Cross-Reference system. It is a Flutter application that lets technicians search a pre-built SQLite database (`parts.db`) of Canon copier parts, cross-reference them across model series, and share selected parts lists as PDFs.

> ⚠️ **This is not the original desktop GUI (Wails) application.** The PDF parser and database builder tool is maintained in a separate repository. This repo contains only the Android mobile app.

---

## 🚀 Features
- **Smart Search**: Search by part number (with/without hyphens) or keyword across description and remarks.
- **Cross-Reference**: See every model series a part appears in, with figure ID and key number.
- **Parts Share**: Select rows and export a formatted PDF parts list via the system share sheet.
- **DB Info Footer**: Shows the database last-updated timestamp and total part count.
- **Update Database**: Download an updated `parts.db` directly from a GitHub release.

## 📖 Build Instructions

### ⚠️ Before building, ensure `parts.db` is up to date!
Copy the latest `parts.db` (generated by the separate CanonBow builder tool) into:
```
bow_mobile/assets/parts.db
```
The bundled DB is what ships inside the APK. If you skip this step, users will receive stale data.

### Development
```bash
cd bow_mobile
flutter pub get
flutter run
```

### Release APK
```bash
cd bow_mobile
flutter pub get
# !! REMINDER: Place an updated parts.db in bow_mobile/assets/ before this step !!
flutter build apk --release
```
The APK is output to `bow_mobile/build/app/outputs/flutter-apk/app-release.apk`.

---

## 🔄 Updating the Database (from GitHub)

The app includes a built-in **Update Database** feature. Open the menu (⋮) on the search screen and tap **"Update DB from GitHub"**. Enter the raw GitHub URL to a `parts.db` file (e.g. a GitHub release asset URL) and the app will download and apply it immediately.

## 📝 Developer Notes
- **Database Schema**: See [`DATABASE_SCHEMA.md`](DATABASE_SCHEMA.md) for the full SQLite schema.
- **Search Logic**: Normalizes part numbers by stripping hyphens; searches `part_number`, `base_part`, `description`, and `remarks`.
- **DB Update**: Downloaded databases are saved to the app's documents directory and replace the bundled copy for subsequent launches.
